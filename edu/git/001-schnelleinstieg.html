Schnelleinstieg

Git ist eine Software, mit der sich verschiedene Versionen von Code einfach organisieren lassen.
Außerdem können durch Git viele Programmierer gemeinsam an einem Projekt arbeiten, ohne große schwierigkeiten zu bekommen.

Starten wir, indem wir ein Repository (kurz "Repo"), also einen Speicherort für unser Projekt erstellen.
Dazu begeben wir uns zuerst in den Ordner auf dem Computer, in dem das Projekt liegt und erstellen das Repo mit: 


git init

Nun haben wir ein Git Repo erstellt. Du kannst jetzt Dateien erstellen oder mit bereits existierenden Dateien fortfahren.
Diese Dateien müssen nun dem Repo hinzugefügt und für den ersten Commit (Snapshot) gestaged werden. Das geht mit: 


git add [Dateiname]

Oder für alle Dateien im Ordner und allen Unterordnern:


git add .

Jetzt sind diese Dateien dem Repo hinzugefügt und gestaged, allerdings ist ihr Zustand noch nicht gespeichert. Wenn du die Änderungen in einer Datei zu einem Commit (Also Snapshot des Codes) hinzufügen willst musst du sie jedes mal vor dem neuen Commit mit "git add" Stagen.
Wenn du meinst, dass ein Zeitpunkt für einen Snapshot, also eine Sicherung vom Dateizustand, gekommen ist, machst du diesen mit: 


git commit -m "Was du getan hast"

Die Beschreibung, was du getan hast, wird neben deinem Snapshot mitgespeichert, damit du besser weist, wofür der jeweilige Snapshot genau war.
Neben der kurzen Beschreibung deiner Änderungen am Code, bekommt der Snapshot auch eine ID zugewiesen.
Der Commit den du gerade erstellt hast, ist nun der head (also neuste Dateizustand) im Master (oder Main) Branch. Nun fragst du dich, was ein Branch ist. Die Entwicklung von Software verläuft nicht geradlinig, sondern sehr verstrickt, da mehrere Entwickler gleichzeitig am Code arbeiten. Damit diese nun nicht gegenseitig ihren Code zerstören, gibt es Branches.
Wenn du einen neuen Branch erstellst, erstellst du ein Komplettes Abbild des Codes, mit dem nur du arbeitest ohne, dass irgendjemand was ungewolltes tut. Du erstellst einen Branch mit: 


git branch [Irgendein Name]

Standartmäßig bist du noch im master (oder main) Branch.
Nun wechselst du also in den Branch, den du gerade erstellt hast mit: 


git checkout [Der Name vom Branch]

Hier kannst du jetzt an Dateien Programmieren und alles mit "git add" und "git commit" versionieren, ohne Einfluss auf dem master branch zu haben.

Wenn das für immer so wäre, wäre das natürlich Sinnlos, also willst du die veränderungen, die du gemacht hast, irgendwann wieder in den master branch übertragen.
Zuerst machts du dafür: 


git checkout master

Um in den Masterbranch zu wechseln und dann: 


git merge [name des Branches, in dem du gearbeitet hast]

Um die Änderungen in den Master Branch zu übertragen.

Nun kannst du entweder mit "git checkout" zurück in den Branch gehen an dem du gearbeitet hast, oder diesen löschen mit: 


git branch -d [Name des zu löschenden Branches]
