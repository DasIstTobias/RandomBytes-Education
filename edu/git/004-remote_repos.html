Remote Repos

Alles was hier ich hier bisher erklärt habe, war nur lokal. Wenn du mit einem Team an einem großen Projekt arbeitest ist das aber wahrscheinlich nicht nur lokal, damit von verschiedenen Orten aus gearbeitet werden kann.
Um remote zu arbeiten benötigt man einen Git Server. Der bekannteste ist wahrscheinlich Github, aber es gibt auch andere welche man sich auch teilweise selber hosten kann, wie Gitlab, Codeberg oder Gitea.

Zuerst musst du ein Remote Repository erstellen falls du das noch nicht getan hast oder es noch keines gibt. Wie das geht ist je nach Service unterschiedlich, aber wenn du es geschafft hat, solltest du einen https oder ssh link bekommen. Ich werde hier HTTPS nutzen.
Wenn es noch keinen Code gibt, kannst du erstmal mit einem offline Repository beginnen. Wenn du fertig bist, kannst du über: 


git remote add origin https://domain.tld/deinName/Repostiory.git

das das standard remote Repository festlegen, welches dann unter dem namen "origin" in deinem offline Repository vermerkt ist.
nun kannst du den Code deines aktuellen Branches über: 


git push origin [Remote Branch name]

in den von dir angegebenen Branch im Remote Repository pushen. Ich würde es an dieser stelle erstmal mit "git push origin master" in den Remote master Branch pushen.

Natürlich geht das nicht einfach so. Bevor du das machst, musst du dich authentifizieren, um dem Server zu zeigen, dass du berechtigungen hast, am Remote Repository zu Arbeiten. Das geht standartmäßig über ssh schlüssel, Access Token oder Nutzername und Passwort, aber manche dienste wie Github bieten auch andere möglichkeiten sich zu Authentifizieren.
Außerdem brauchst du einen Namen und eine Email Adresse, die du angeben kannst. Diese setzt du dann bei dir lokal (und am besten gleich auf dem ganzen Gerät) im mit: 


git config --global user.name "Dein Name"

und: 


git config --global user.email "email@example.com"

Wenn du schon ein Repository mit Code haben solltest, bei dem du anfängst mit zu entwickeln, oder das du auf ein neues Gerät übertragen willst, kannst du es mit: 


git clone https://domain.tld/deinName/Repostiory.git

auf dein Gerät klonen. Dies tust du in einen leeren Ordner, der noch kein Repository enthält, da beim klonen auch das Repository selbst vom Server gezogen wird. Außerdem musst du erstmal kein remote Repository setzen, da origin automatisch auf die Adresse gesetzt wird, von der du geklont hast.
Wenn du schon ein Repository auf deinem Gerät hast, aber das Repository auf dem Server aktuellere Commits hat, kannst du die Änderungen auf dein Gerät synchronisieren. Das geht mit: 


git fetch origin
